## 核心概念

### 1. Agent vs Workflow

#### Workflow (Chain)
- **定义**: 预定义的固定步骤流程
- **特点**: 确定性、可预测、高效
- **适用**: 固定流程、批量处理、ETL任务

#### Agent
- **定义**: 能够自主决策和使用工具的智能系统
- **特点**: 动态决策、自主规划、灵活适应
- **适用**: 需要工具调用、动态决策、复杂任务

#### 选择原则
```
简单任务 → 直接API调用
固定流程 → Workflow
动态决策 → Agent
```

### 2. LangChain核心组件

#### Models
- 统一的LLM接口
- 支持多种提供商（OpenAI, Anthropic等）
- 可配置参数（temperature, max_tokens等）

#### Prompts
- `PromptTemplate`: 简单模板
- `ChatPromptTemplate`: 对话模板
- `FewShotPromptTemplate`: Few-shot学习

#### Chains
- `LLMChain`: 基础链
- `SequentialChain`: 顺序执行多个步骤
- `RouterChain`: 条件路由

#### Agents
- `ReAct Agent`: Reasoning + Acting
- `OpenAI Functions Agent`: 使用函数调用
- `Structured Chat Agent`: 结构化对话

#### Tools
- 工具定义和注册
- 清晰的描述（影响Agent选择）
- 错误处理

#### Memory
- `ConversationBufferMemory`: 保存所有历史
- `ConversationBufferWindowMemory`: 滑动窗口
- `ConversationSummaryMemory`: 摘要模式
- `ConversationSummaryBufferMemory`: 混合模式

### 3. Agent工作原理

#### ReAct模式
```
1. Thought: 思考要做什么
2. Action: 选择工具
3. Action Input: 工具输入
4. Observation: 工具输出
5. 重复1-4直到得到答案
6. Final Answer: 最终答案
```

#### 关键参数
- `verbose`: 显示推理过程
- `max_iterations`: 最大迭代次数（防止无限循环）
- `handle_parsing_errors`: 自动处理解析错误
- `early_stopping_method`: 提前停止策略

### 4. 工具开发最佳实践

#### 工具设计原则
1. **单一职责**: 每个工具只做一件事
2. **清晰描述**: 详细说明工具功能和输入格式
3. **错误处理**: 完善的异常处理
4. **类型提示**: 使用类型注解

#### 工具描述示例
```python
Tool(
    name="Calculator",
    func=calculate,
    description="""用于数学计算。支持基本运算和数学函数。
输入应该是一个数学表达式，例如: '2+2', 'sqrt(16)'"""
)
```

### 5. Memory选择指南

| Memory类型 | 适用场景 | Token消耗 | 优缺点 |
|-----------|---------|----------|--------|
| BufferMemory | 短对话(<10轮) | 高 | 完整历史，成本高 |
| WindowMemory | 中等对话(10-50轮) | 中 | 可控成本，丢失早期信息 |
| SummaryMemory | 长对话(>50轮) | 低 | 节省token，可能丢失细节 |
| SummaryBufferMemory | 长对话 | 中 | 平衡性能和上下文 |

### 6. 框架对比

#### LangChain
- **优势**: 组件丰富、生态完善、社区活跃
- **劣势**: 学习曲线陡、抽象层次多
- **适用**: 通用Agent开发、复杂系统

#### LlamaIndex
- **优势**: 专注RAG、检索质量高、API简单
- **劣势**: 功能相对单一
- **适用**: 文档问答、知识库检索

#### AutoGPT
- **优势**: 高度自主、任务分解强
- **劣势**: 成本高、不稳定
- **适用**: 研究实验、自主任务

#### CrewAI
- **优势**: 多Agent协作、角色清晰
- **劣势**: 生态较小、相对新
- **适用**: 多角色协作、团队模拟

### 7. 成本控制策略

#### Workflow成本
- 固定成本 = 步骤数 × 每步token
- 可预测、可控制

#### Agent成本
- 变动成本 = 迭代次数 × 每次token
- 设置`max_iterations`限制
- 使用缓存减少重复调用
- 选择合适的模型（如gpt-4o-mini）

### 8. 调试技巧

#### Workflow调试
1. 检查每个步骤的输出
2. 使用`verbose=True`
3. 单独测试每个Chain

#### Agent调试
1. 启用`verbose=True`查看推理过程
2. 检查工具描述是否清晰
3. 查看`intermediate_steps`
4. 单独测试每个工具
5. 使用callbacks记录日志

### 9. 常见问题

#### Agent陷入循环
- 原因: 工具描述不清晰、任务太复杂
- 解决: 优化工具描述、设置max_iterations、简化任务

#### 工具选择错误
- 原因: 工具描述不准确
- 解决: 改进描述、提供示例、减少工具数量

#### Memory占用过多
- 原因: 使用BufferMemory且对话很长
- 解决: 切换到WindowMemory或SummaryMemory

#### 成本过高
- 原因: 过多的LLM调用、使用昂贵模型
- 解决: 使用Workflow代替Agent、选择便宜模型、设置max_iterations

## 实践经验

### 1. 何时使用Agent
- ✅ 需要工具调用
- ✅ 用户意图不明确
- ✅ 需要动态决策
- ❌ 固定流程（用Workflow）
- ❌ 成本敏感（用Workflow）

### 2. 工具开发经验
- 工具描述要详细，包含输入格式示例
- 每个工具只做一件事
- 完善的错误处理和返回信息
- 测试工具的边界情况

### 3. Prompt优化
- System message设定角色和行为
- 提供Few-shot示例
- 明确输出格式要求
- 使用结构化输出

### 4. 生产环境建议
- 优先使用Workflow
- Agent要设置max_iterations
- 实现重试机制
- 记录详细日志
- 监控成本和性能

## 代码示例总结

### 基础Agent示例
```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_react_agent, Tool
from langchain import hub

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

tools = [
    Tool(name="Calculator", func=calculate, description="计算数学表达式")
]

prompt = hub.pull("hwchase17/react")
agent = create_react_agent(llm, tools, prompt)

agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=5
)

result = agent_executor.invoke({"input": "计算 2+2"})
```

### Workflow示例
```python
from langchain.chains import LLMChain, SequentialChain
from langchain.prompts import PromptTemplate

chain1 = LLMChain(llm=llm, prompt=prompt1, output_key="output1")
chain2 = LLMChain(llm=llm, prompt=prompt2, output_key="output2")

workflow = SequentialChain(
    chains=[chain1, chain2],
    input_variables=["input"],
    output_variables=["output1", "output2"]
)

result = workflow.invoke({"input": "..."})
```

### Memory示例
```python
from langchain.memory import ConversationBufferWindowMemory

memory = ConversationBufferWindowMemory(k=5)

conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=True
)
```
