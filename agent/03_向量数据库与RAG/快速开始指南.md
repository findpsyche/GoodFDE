## 🚀 快速开始

### 步骤1: 安装依赖

```bash
cd 03_向量数据库与RAG
pip install -r requirements.txt
```

**主要依赖包**：
- `langchain` - 核心框架
- `langchain-openai` - OpenAI集成
- `chromadb` - 向量数据库
- `faiss-cpu` - 高性能向量检索
- `sentence-transformers` - 开源Embedding模型
- `pypdf` - PDF文档处理

**安装时间**: 约2-3分钟

### 步骤2: 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件
# OPENAI_API_KEY=your_key_here
```

## 🐛 常见问题排查

### 问题1: 安装chromadb失败
```bash
# 错误信息
ERROR: Could not build wheels for chromadb
```

**解决方案**：
```bash
# 方案1: 升级pip
pip install --upgrade pip setuptools wheel

# 方案2: 使用预编译版本
pip install chromadb --no-build-isolation

# 方案3: 使用conda
conda install -c conda-forge chromadb
```

---

### 问题2: FAISS安装问题
```bash
# 错误信息
No module named 'faiss'
```

**解决方案**：
```bash
# CPU版本
pip install faiss-cpu

# GPU版本（需要CUDA）
pip install faiss-gpu
```

---

### 问题3: 向量数据库不存在
```bash
# 错误信息
⚠️ 请先运行 04_basic_rag.py 创建向量数据库
```

**解决方案**：
```bash
# 先运行基础RAG脚本创建数据库
python 04_basic_rag.py

# 然后再运行其他脚本
python 05_advanced_rag.py
```

---

### 问题4: 检索结果为空
**原因**：
- 文档未正确加载
- Embedding模型不匹配
- 相似度阈值过高

**解决方案**：
1. 检查文档是否存在
2. 确认embedding模型一致
3. 降低相似度阈值
4. 增加top_k数量

---

### 问题5: 答案质量差
**原因**：
- chunk_size不合适
- 检索文档不相关
- Prompt模板不佳

**解决方案**：
1. 调整chunk_size (尝试300-800)
2. 使用重排序
3. 优化Prompt模板
4. 增加检索文档数

---

### 问题6: 成本过高
**原因**：
- 使用昂贵的模型
- 过多的API调用
- 没有缓存

**解决方案**：
1. 使用gpt-4o-mini
2. 使用开源embedding
3. 实施缓存策略
4. 上下文压缩
