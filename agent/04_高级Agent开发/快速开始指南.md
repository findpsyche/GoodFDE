## 🎯 核心概念速览

### ReAct模式

```
用户问题
    ↓
Thought: 我需要做什么？
    ↓
Action: 调用工具
    ↓
Observation: 观察结果
    ↓
Thought: 结果如何？继续还是结束？
    ↓
[循环 或 返回答案]
```

**关键点**：
- 每一步都有明确的思考过程
- 工具调用基于推理
- 可以多轮迭代

### Multi-Agent协作

```
任务
  ↓
协调员
  ├─→ 研究员 → 收集信息
  ├─→ 编辑   → 整理内容
  └─→ 审核员 → 验证质量
  ↓
汇总结果
```

**关键点**：
- 专业分工
- 协调机制
- 迭代改进

## 🔍 设计层次对照

### ❌ 下限：碰运气完成任务

```python
# 只管调用，不管验证
agent = Agent(tools=[search, calculator])
result = agent.run("帮我研究Python最佳实践")
print(result)  # 真的完成了吗？质量如何？
```

**问题**：
- 不知道Agent做了什么
- 不知道结果是否正确
- 出错了不知道怎么办

### ✅ 及格线：能观察、能诊断

```python
# 有日志、验证、错误处理
agent = Agent(
    tools=[search, calculator],
    verbose=True,  # 启用详细日志
    max_iterations=10  # 避免无限循环
)

try:
    result = agent.run("帮我研究Python最佳实践")

    # 验证结果
    if len(result) < 100:
        raise ValueError("结果太短，可能不完整")

    print(f"✅ 完成！结果长度: {len(result)}")

except Exception as e:
    print(f"❌ 失败: {e}")
    # 查看日志定位问题
```

**改进**：
- 能看到Agent的每一步
- 能验证结果质量
- 能处理错误

### 🌟 状元：能闭环、能迭代

```python
# 闭环验证，自动改进
def research_with_verification(topic, max_attempts=3):
    for attempt in range(max_attempts):
        # 执行研究
        result = agent.run(f"研究{topic}")

        # 评估质量
        score = evaluate_quality(result)

        if score >= 8.0:
            return result

        # 质量不够，分析问题
        issues = analyze_issues(result)

        # 改进prompt
        topic = improve_prompt(topic, issues)

        print(f"尝试 {attempt + 1}: 质量 {score}/10，继续改进...")

    raise Exception("未能达到质量要求")
```

**改进**：
- 有质量评估
- 能自动改进
- 有完整闭环

## ❌ 遇到问题？

### 问题1：找不到模块

```
ModuleNotFoundError: No module named 'langchain'
```

**解决方案**：
```bash
pip install -r requirements.txt
```

### 问题2：API密钥错误

```
Error: Invalid API key
```

**解决方案**：
1. 检查 `.env` 文件中的密钥是否正确
2. 确保没有多余的空格
3. 确保密钥以 `sk-` 或 `sk-ant-` 开头

### 问题3：Agent陷入循环

```
Agent执行超过最大迭代次数
```

**解决方案**：
1. 检查工具描述是否清晰
2. 增加 `max_iterations` 参数
3. 改进prompt，提供更明确的指令

### 问题4：工具调用失败

```
Tool execution failed: ...
```

**解决方案**：
1. 查看详细错误信息
2. 检查工具参数是否正确
3. 添加错误处理和降级策略
