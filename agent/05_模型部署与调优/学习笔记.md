### 1.2 vLLM 高性能部署

#### 核心概念

**PagedAttention**
- 将 KV Cache 分页管理
- 类似操作系统的虚拟内存
- 减少内存碎片
- 提高内存利用率

**连续批处理（Continuous Batching）**
```
传统批处理:
Batch 1: [Req1, Req2, Req3] → 等待最慢的完成
Batch 2: [Req4, Req5, Req6] → 等待最慢的完成

连续批处理:
[Req1, Req2, Req3] → Req2 完成 → 加入 Req4
[Req1, Req3, Req4] → Req1 完成 → 加入 Req5
```

## 第二部分：模型量化

### 2.1 量化原理

#### 核心概念

**什么是量化？**
- 将高精度数值转换为低精度
- FP16 (16位浮点) → INT8 (8位整数) → INT4 (4位整数)
- 减少模型大小和内存占用
- 加快推理速度

**量化方法**：
1. **权重量化**：只量化模型权重
2. **激活量化**：同时量化激活值
3. **动态量化**：运行时量化
4. **静态量化**：预先量化

#### 量化格式对比

| 格式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| GGUF | 通用、支持CPU | 精度稍低 | 本地部署 |
| GPTQ | GPU快、精度高 | 需要GPU | GPU推理 |
| AWQ | 精度最高 | 速度稍慢 | 质量优先 |

## 第三部分：模型微调

### 3.1 LoRA 原理

#### 核心概念

**LoRA (Low-Rank Adaptation)**
```
原始权重矩阵: W (d × d)
LoRA 分解: W + ΔW = W + BA
  其中 B (d × r), A (r × d), r << d

参数量: d² → 2dr (大幅减少)
```

**为什么有效？**
- 模型适应新任务只需要低秩更新
- 冻结原始权重，只训练 LoRA 参数
- 可以快速切换不同的 LoRA 适配器

#### QLoRA 优势

```
LoRA: 在 FP16 模型上训练
QLoRA: 在 INT4 模型上训练

内存需求:
  7B FP16: ~14GB
  7B INT4 + LoRA: ~6GB

可以在消费级 GPU 上微调 70B 模型！
```

### 4.1 批处理优化

#### 核心概念

**批处理的权衡**：
```
Batch Size ↑
  → 吞吐量 ↑ (更多请求/秒)
  → 延迟 ↑ (单个请求等待时间)
  → 内存占用 ↑
```

### 4.2 KV Cache 优化

#### 核心概念

**KV Cache 是什么？**
- 缓存已计算的 Key 和 Value
- 避免重复计算
- 内存占用：O(batch_size × seq_len × hidden_dim)

**优化策略**：
1. 限制最大序列长度
2. 使用滑动窗口
3. PagedAttention（vLLM）

### 4.3 Flash Attention

#### 核心概念

**Flash Attention 原理**：
- 优化内存访问模式
- 减少 HBM ↔ SRAM 数据传输
- 速度提升 2-4 倍

**硬件要求**：
- NVIDIA Ampere 架构以上
- A100, RTX 3090, RTX 4090 等
