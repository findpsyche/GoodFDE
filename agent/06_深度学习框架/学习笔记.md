### 1.2 自动微分

#### 核心概念
- `requires_grad=True` 开启梯度追踪
- `loss.backward()` 反向传播
- `optimizer.step()` 更新参数
- `optimizer.zero_grad()` 清零梯度

**为什么要清零梯度？**
- PyTorch 默认累积梯度
- 每次反向传播前必须手动清零
- 否则梯度会叠加导致错误更新

### 1.3 神经网络模块（nn.Module）

```python
        x = self.layer2(x)
        return x
```

## 第二部分：Transformer 架构

### 2.1 Attention 机制

#### 核心概念
- Query, Key, Value 三个矩阵
- Scaled Dot-Product Attention: `Attention(Q,K,V) = softmax(QK^T/√d_k)V`
- 为什么要 scale：防止 softmax 梯度消失

**直观理解**
- Query：我想找什么
- Key：我是什么
- Value：我的内容是什么
- Attention 权重：Query 和 Key 的相似度

### 2.3 Position Encoding

#### 核心概念
- Transformer 没有位置信息，需要额外编码
- 正弦/余弦位置编码
- 可学习的位置编码

**正弦位置编码公式**
```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

## 第四部分：训练技巧

### 4.1 学习率调度

#### 核心概念
- Warmup + Decay
- Cosine Annealing
- Linear Decay

**为什么需要 Warmup？**
- 训练初期参数随机，梯度不稳定
- 小学习率让模型先适应数据
- 然后逐渐增大到目标学习率

**常用调度器**
```python
from transformers import get_scheduler

scheduler = get_scheduler(
    "linear",
    optimizer=optimizer,
    num_warmup_steps=500,
    num_training_steps=10000
)
```

### 4.2 梯度累积

#### 核心概念
- 模拟大 batch size
- 每 N 步才更新参数
- 适用于显存不足的情况

**实现方式**
```python
accumulation_steps = 4

for i, batch in enumerate(dataloader):
    loss = model(batch)
    loss = loss / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

### 4.3 混合精度训练

#### 核心概念
- FP16 + FP32 混合
- 减少显存占用
- 加速训练

**使用 PyTorch AMP**
```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for batch in dataloader:
    optimizer.zero_grad()

    with autocast():
        loss = model(batch)

    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

### 5.2 诊断训练问题

#### 常见问题速查表

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| Loss 不下降 | 学习率太小/数据问题 | 增大学习率/检查数据 |
| Loss 爆炸 | 学习率太大 | 减小学习率/梯度裁剪 |
| Loss 震荡 | 学习率太大/batch size 太小 | 减小学习率/增大 batch |
| 过拟合 | 模型太大/数据太少 | 正则化/数据增强/Dropout |
| 欠拟合 | 模型太小/训练不足 | 增大模型/增加训练轮数 |
| 梯度消失 | 网络太深/激活函数问题 | 残差连接/换激活函数 |
| 梯度爆炸 | 学习率太大/初始化问题 | 梯度裁剪/调整初始化 |
| OOM (显存不足) | Batch size 太大 | 减小 batch/梯度累积/混合精度 |

#### 调试技巧

**1. 先过拟合小数据集**
- 取 10-100 条数据
- 确保模型能完全记住
- 验证模型和训练流程正确

**2. 检查数据**
- 打印几个样本
- 检查标签分布
- 验证预处理正确

**3. 监控梯度**
```python
for name, param in model.named_parameters():
    if param.grad is not None:
        print(f"{name}: {param.grad.norm()}")
```

**4. 使用梯度裁剪**
```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```
